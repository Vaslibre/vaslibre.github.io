La última versión del popular servidor Web Java Open Source está disponible desde ayer a través de la Fundación Apache, y cuenta con mejoras en el soporte de Java Servlet, JSP 2.2 y especificaciones EL 2.2.

[![tomcat](http://tomcat.apache.org/images/tomcat10.jpg)](http://tomcat.apache.org/)

Apache Tomcat 7 permite escribir y desplegar aplicaciones Web complejas de forma sencilla, proporcionando el soporte para características de desarrollo que de otra forma tendrían que ser desplegadas manualmente. La nueva versión resuelve de forma más eficiente todo lo que son las opciones de autenticación de usuarios, los registros de log, las herramientas de fortificación del servidor, así como la infraestructura y los filtrados a nivel de aplicación. La fortificación de un servidor Web típico tiene muchos detalles que deben tenerse en cuenta, ya que es uno de los servidores más expuestos de las organizaciones.

 

Con más de 10 millones de descargas llevadas a cabo de sus versiones anteriores, Apache Tomcat es desplegado actualmente en un amplio rango de Websites, incluyendo muchos de misión crítica, entre las que se encuentran algunas grandes corporaciones que forman parte de la lista Fortune 500, proporcionando servicio a sistemas eCommerce. MuleSoft es una de las compañías que destaca el desarrollo de Tomcat 7, según destaca Ross Mason, fundador y CTO de MuleSoft. Las novedades introducidas permiten aprovechar los avances en Java, incluyendo la especificación Servlet 3, y mejorando el ciclo de vida de los desarrollos de aplicaciones Web.

[![apache](http://www.apache.org/images/asf-logo.gif)](http://www.apache.org)

Todo el conglomerado de desarrolladores que utilizan Tomcat 7 también va a beneficiarse de la mejora, corrigiendo errores, así como actualizaciones para afrontar determinadas vulnerabilidades de seguridad. Los programadores y diseñadores de páginas web suelen utilizar un escenario alternativo a su servidor para poder realizar pruebas, actualizar contenidos o mejorar diseños. Las versiones anteriores Tomcat 5.5 y 6.0 seguirán disponiendo del correspondiente soporte.

Sitio web oficial: [http://tomcat.apache.org/](http://tomcat.apache.org/)
